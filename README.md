# Искра 🔥 — Telegram-бот знакомств

Бот знакомств в духе «Дайвинчика», но с фишками, которые делают знакомства живее.
Написан на **aiogram 3** + **SQLite** (aiosqlite). Готов к деплою на **Railway** и **Viktor.com**.

[![Deploy on Railway](https://railway.com/button.svg)](https://railway.com/new?referralCode=&template=https%3A%2F%2Fgithub.com%2Fmrslava777%2Fiskra-bot)

> Нажми кнопку → войди через GitHub → выбери репозиторий `iskra-bot` → в разделе **Variables** добавь `BOT_TOKEN` → **Deploy**.

## Деплой на Railway
1. Создай проект из этого репозитория (auto-deploy на push).
2. В Variables добавь `BOT_TOKEN` (и при желании `ADMIN_IDS`).
3. Для сохранения базы между деплоями примонтируй Volume и задай `DB_PATH=/data/iskra.db`.
4. Стартовая команда: `python bot.py` (уже в `railway.json` / `Procfile`).

## Деплой на Viktor.com
1. Подключи репозиторий `mrslava777/iskra-bot`.
2. Укажи `BOT_TOKEN` в переменных окружения.
3. Запусти — бот автоматически поднимется.

## Структура
Проект разбит по слоям: точка входа → конфиг → доступ к данным → бизнес-логика
(`services`) → представление (`handlers`) → статический контент (`data`).
```
bot.py              # точка входа: запуск (polling/webhook), middleware, обработка ошибок
config.py           # переменные окружения, авто-путь к БД
database.py         # слой доступа к SQLite: пользователи, лайки, мэтчи, жалобы, значки
states.py           # FSM-состояния (регистрация, редактирование, верификация, поддержка)
keyboards.py        # клавиатуры/кнопки
health.py           # /health-эндпоинт для мониторинга
handlers/           # обработчики апдейтов (по одному файлу на раздел)
  start.py          # регистрация анкеты
  browse.py         # лента, лайки, мэтчи
  likes.py          # входящие симпатии и список мэтчей
  profile.py        # своя анкета, вопрос дня, настройки
  badges.py         # 🏆 значки
  relationships_handler.py  # уровень отношений в анон-чате
  anon.py           # свидание вслепую
  support.py        # тикеты поддержки
  admin.py          # админ-панель
  misc.py           # /stats и фолбэк
  __init__.py       # сборка роутеров (setup_routers)
services/           # бизнес-логика (без обращения к Telegram-слою)
  matching.py       # совместимость, оформление анкет
  badges.py         # логика значков
  relationships.py  # начисление очков/уровней отношений
middlewares/        # сквозные обработчики
  throttling.py     # анти-флуд (ограничение частоты сообщений)
data/               # статический контент
  content.py        # интересы, вопросы дня, айсбрейкеры
  badges.py         # определения значков
```

## ⚙️ Масштабирование
По умолчанию бот работает как раньше: **long polling + in-memory FSM + SQLite**.
Этого с запасом хватает на десятки тысяч пользователей. Для роста ничего в коде
менять не нужно — достаточно переменных окружения (каждая опциональна):

| Переменная | Эффект |
|------------|--------|
| `REDIS_URL` | FSM хранится в Redis (общее состояние для нескольких инстансов, переживает рестарт). Без неё — in-memory. |
| `WEBHOOK_URL` | Бот работает через webhook вместо polling (нужно для нескольких инстансов за балансировщиком). Доп.: `WEBHOOK_PATH`, `WEBHOOK_SECRET`. Без неё — polling. |

Анти-флуд middleware включён всегда (ограничивает частоту входящих сообщений).

**Путь дальнейшего роста (100k–1M):** вынести БД из SQLite в управляемый
PostgreSQL (SQLite — один писатель на процесс), включить `REDIS_URL` + `WEBHOOK_URL`
и поднять несколько инстансов. Миграция БД — отдельная задача, делается осознанно.

## Мониторинг (`/health`)

Бот поднимает крошечный healthcheck-эндпоинт (модуль `health.py`) в том же
процессе, что и поллинг — чтобы за ботом можно было следить снаружи.

- `GET /health` → `200 {"status":"ok",...}` пока бот жив; `503 {"status":"stale"}`
  если event loop завис; **connection refused** если процесс упал.
- Порт берётся из переменной `PORT` (Railway задаёт её сам).
- Чтобы пинговать снаружи: в Railway у сервиса включи **Public Networking**
  (Settings → Networking → Generate Domain) и используй `https://<домен>/health`.

Логику бота модуль не меняет и при своей ошибке бота не роняет.
