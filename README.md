# Искра 🔥 — Telegram-бот знакомств

Бот знакомств в духе «Дайвинчика», но с фишками, которые делают знакомства живее.
Написан на **aiogram 3** + **SQLite** (aiosqlite). Готов к деплою на **Railway**.

[![Deploy on Railway](https://railway.com/button.svg)](https://railway.com/new?referralCode=&template=https%3A%2F%2Fgithub.com%2Fmrslava777%2Fiskra-bot)

> Нажми кнопку → войди через GitHub → выбери репозиторий `iskra-bot` → в разделе **Variables** добавь `BOT_TOKEN` → **Deploy**.

## Чем интереснее обычных ботов

| Фича | Что делает |
|------|------------|
| 💞 **Совместимость %** | Считает процент совпадения по интересам и рисует прогресс-бар прямо в анкете |
| 🏷 **Интересы (теги)** | До 5 тегов из 20 категорий — основа для матчинга |
| 🎯 **Вопрос дня** | Каждый день новый вопрос; ответ показывается в анкете и выделяет тебя |
| 💬 **Айсбрейкеры** | При мэтче бот предлагает первую фразу, чтобы не было неловкой паузы |
| 💌 **«Кто меня лайкнул»** | Отдельная лента входящих симпатий с лайком в ответ |
| 🔥 **Огоньки и рейтинг** | Анкета набирает симпатии → растёт «огонёк»; стрик за ежедневную активность |
| 🎚 **Умные фильтры** | Фильтр по полу, возрасту, городу + кнопка скрыть анкету |
| 🚩 **Модерация** | Жалобы; после 5 — авто-бан, чтобы лента оставалась чистой |

## Возможности (как у Дайвинчика)
- Регистрация анкеты: имя, возраст, пол, кого ищет, город, фото, о себе
- Лента анкет по одной
- Лайк / дизлайк / лайк с подсказкой сообщения
- Взаимный лайк → мэтч и обмен контактами (@username)
- Просмотр и редактирование своей анкеты

## Запуск локально
```bash
pip install -r requirements.txt
cp .env.example .env   # вставь BOT_TOKEN от @BotFather
python bot.py
```

## Деплой на Railway
1. Создай проект из этого репозитория (auto-deploy на push).
2. В Variables добавь `BOT_TOKEN` (и при желании `ADMIN_IDS`).
3. Для сохранения базы между деплоями примонтируй Volume и задай `DB_PATH=/data/iskra.db`.
4. Стартовая команда: `python bot.py` (уже в `railway.json` / `Procfile`).

## Структура
```
bot.py              # точка входа, polling
config.py           # переменные окружения
database.py         # SQLite: пользователи, лайки, мэтчи, жалобы
states.py           # FSM-состояния
keyboards.py        # клавиатуры
handlers/
  start.py          # регистрация анкеты
  browse.py         # лента, лайки, мэтчи
  likes.py          # входящие симпатии и список мэтчей
  profile.py        # своя анкета, вопрос дня, настройки
  misc.py           # /stats и фолбэк
services/matching.py# совместимость, оформление анкет
data/content.py     # интересы, вопросы дня, айсбрейкеры
```

## Команды
- `/start` — меню / регистрация
- `/myprofile` — моя анкета
- `/help` — помощь
- `/stats` — статистика (только для ADMIN_IDS)

## Мониторинг (`/health`)

Бот поднимает крошечный healthcheck-эндпоинт (модуль `health.py`) в том же
процессе, что и поллинг — чтобы за ботом можно было следить снаружи.

- `GET /health` → `200 {"status":"ok",...}` пока бот жив; `503 {"status":"stale"}`
  если event loop завис; **connection refused** если процесс упал.
- Порт берётся из переменной `PORT` (Railway задаёт её сам).
- Чтобы пинговать снаружи: в Railway у сервиса включи **Public Networking**
  (Settings → Networking → Generate Domain) и используй `https://<домен>/health`.

Логику бота модуль не меняет и при своей ошибке бота не роняет.
